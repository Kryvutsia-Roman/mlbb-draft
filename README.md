# 🎮 MLBB Draft Helper (React Application)

Веб-застосунок для допомоги у фазі драфту (вибору героїв) у грі Mobile Legends: Bang Bang. Застосунок аналізує обраних ворожих та союзних героїв і за допомогою спеціального алгоритму рекомендує найкращі контрпіки та синергії.

Розроблено в рамках **Лабораторної роботи №5 (React Application)**.

## ✨ Основний функціонал

* **Інтерактивний вибір команд:** Формування команди союзників (5 слотів) та команди ворогів (5 слотів).
* **Система рекомендацій (Smart Draft):** Автоматичний розрахунок "ваги" кожного героя на основі локальної бази даних. Алгоритм враховує:
  * Контрпіки ворогів (+ бали).
  * Вразливості до ворогів (- бали).
  * Синергію з уже обраними союзниками (+ бали).
* **Захист від дублювання ролей:** Вбудована бізнес-логіка (попередження), яка запобігає випадковому вибору кількох героїв однакової ролі (наприклад, двох Стрільців або двох Магів).
* **Інтуїтивний UI:** Додавання героя в союзну команду лівим кліком, у ворожу — правим кліком миші.

## 🛠 Стек технологій

* **Frontend:** React (Функціональні компоненти, Hooks)
* **Збірка:** Vite (швидкий HMR та оптимізований білд)
* **Стилізація:** Чистий CSS3 (Flexbox, CSS Transitions)
* **Архітектура:** Поділ на Smart (розумні) та Dumb (презентаційні) компоненти.

## 📂 Структура проєкту

Проєкт дотримується сучасних стандартів архітектури React-застосунків (Separation of Concerns):

```text
src/
├── components/           # Ізольовані UI-компоненти
│   ├── HeroCard.jsx      # Картка окремого героя
│   ├── HeroPool.jsx      # Загальна сітка всіх доступних героїв
│   ├── Recommendations.jsx # Блок рекомендованих піків
│   └── TeamSection.jsx   # Універсальний контейнер для команд
├── hooks/                
│   └── useDraftLogic.js  # Кастомний хук: інкапсульована бізнес-логіка та стан (State)
├── heroes.js             # Локальна "база даних" (понад 70 героїв, контрпіки, синергії)
├── App.jsx               # Головний компонент-оркестратор
├── main.jsx              # Точка входу застосунку
└── App.css               # Глобальні стилі проєкту

## 📖 Як користуватися застосунком (User Guide)

Після успішного запуску локального сервера (`npm run dev`) та відкриття сторінки у браузері, дотримуйтесь таких кроків для драфту:

1. **Вибір союзників (My Team):** * Наведіть курсор на героя у загальному пулі (нижня частина екрану) та клікніть **ЛІВОЮ кнопкою миші**. 
   * Герой переміститься у синю панель вашої команди.
2. **Вибір ворогів (Enemy Team):** * Наведіть курсор на героя у загальному пулі та клікніть **ПРАВОЮ кнопкою миші**. 
   * Герой переміститься у червону панель ворожої команди.
3. **Аналіз рекомендацій:** * Щойно ви додасте хоча б одного героя на дошку, посередині екрану з'явиться секція **"💡 Рекомендовані піки"**.
   * Програма автоматично вирахує бали і запропонує ТОП-5 найкращих варіантів для вашої команди. Під кожним рекомендованим героєм будуть вказані причини (наприклад, "Контрить Layla" або "Синергія з Tigreal").
4. **Розумне попередження (Role Lock):** * Якщо ви спробуєте додати до своєї команди другого героя такої ж ролі (наприклад, другого Мага), з'явиться діалогове вікно з попередженням про порушення балансу. Ви можете скасувати дію або свідомо підтвердити такий пік.
5. **Видалення та заміна:** * Щоб прибрати героя з драфту (якщо ви помилилися кліком), просто натисніть на його картку у верхній панелі союзників чи ворогів. Він знову стане активним у загальному пулі.
